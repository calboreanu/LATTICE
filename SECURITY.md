# Security Policy

This repository contains a governance engine and validation utilities. If you find a security issue — for example a gate bypass, a fail-open in policy evaluation, an audit-chain or signature-verification bypass, or a coordination-plane authentication weakness — please open a private security report if available, or file an issue with minimal exploit detail.

Do not include real targets, hostnames, or credentials in reports or test vectors; use placeholders (e.g., `acme.example.com`, `10.0.1.0/24`).
